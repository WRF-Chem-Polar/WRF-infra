# Naming conventions

## Issues

Issue titles should be a sentence that start with an action verb, for instance:

 - Add docstrings to functions in myscript.py
 - Remove use of import * in this-piece-of-code.py

## Branches

A branch **must** be attached to an issue. The name of the branch should start with `issue##/` (where `##` is the issue number), followed by a short human-readable description, for instance:

- issue119/dates-to-conf-file
- issue84/add-cloud-properties
